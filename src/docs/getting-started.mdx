# Getting Started with MDX

MDX lets you write **JSX directly inside Markdown**. This means you can use React components alongside standard Markdown syntax — perfect for interactive documentation.

## What is MDX?

MDX combines two things you already know:

- **Markdown** — the simple formatting syntax for text
- **JSX** — React's syntax for rendering components

The result: documentation that is both readable as plain text _and_ interactive when rendered.

## Using JavaScript Expressions

MDX supports JavaScript expressions with curly braces:

The current year is: **{new Date().getFullYear()}**

You can also use expressions inline: there are **{[1, 2, 3, 4, 5].length}** items in this array.

## Code Blocks

Standard fenced code blocks get styled automatically with syntax highlighting:

```jsx
import { Alert } from './components';

function MyPage() {
  return (
    <Alert variant="warning" title="Heads up!">
      This is a custom component in MDX.
    </Alert>
  );
}
```

You can also use the `CodeTabs` component to show the same example in multiple languages:

<CodeTabs tabs={[
  {

}`
  }
