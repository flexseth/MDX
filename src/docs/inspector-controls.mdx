# InspectorControls

A slot component that renders its children into the Settings sidebar — the right-hand panel in the block editor. Mirrors the WordPress [`InspectorControls`](https://developer.wordpress.org/block-editor/reference-guides/components/inspector-controls/) from `@wordpress/block-editor`.

> **Note:** This component requires the WordPress block editor environment and cannot be rendered outside of a WordPress block. The examples below show correct usage in `edit.js`.

## Basic Usage

```jsx
import { InspectorControls } from '@wordpress/block-editor';
import { PanelBody, TextControl, ToggleControl } from '@wordpress/components';

// Inside your Edit component's return, alongside the block content:
<InspectorControls>
  <PanelBody title={ __( 'Settings', 'your-text-domain' ) }>
    <TextControl
      label={ __( 'Title', 'your-text-domain' ) }
      value={ title }
      onChange={ ( value ) => setAttributes( { title: value } ) }
    />
    <ToggleControl
      label={ __( 'Show Border', 'your-text-domain' ) }
      checked={ showBorder }
      onChange={ ( value ) => setAttributes( { showBorder: value } ) }
    />
  </PanelBody>
</InspectorControls>
```

## Common Patterns

### Multiple collapsible panels

Organize settings into multiple `PanelBody` sections to group related controls. Set `initialOpen={ false }` on secondary panels to prevent sidebar clutter when the block is first selected.

```jsx
import { __ } from '@wordpress/i18n';
import { useBlockProps, InspectorControls } from '@wordpress/block-editor';
import {
  PanelBody,
  TextControl,
  ToggleControl,
  RangeControl,
  SelectControl,
} from '@wordpress/components';

export default function Edit( { attributes, setAttributes } ) {
  const blockProps = useBlockProps();
  const { heading, showDate, fontSize, alignment } = attributes;

  return (
    <>
      <InspectorControls>
        <PanelBody
          title={ __( 'Content', 'your-text-domain' ) }
          initialOpen={ true }
        >
          <TextControl
            label={ __( 'Heading', 'your-text-domain' ) }
            value={ heading }
            onChange={ ( value ) => setAttributes( { heading: value } ) }
          />
          <ToggleControl
            label={ __( 'Show Date', 'your-text-domain' ) }
            checked={ showDate }
            onChange={ ( value ) => setAttributes( { showDate: value } ) }
          />
        </PanelBody>

        <PanelBody
          title={ __( 'Typography', 'your-text-domain' ) }
          initialOpen={ false }
        >
          <RangeControl
            label={ __( 'Font Size', 'your-text-domain' ) }
            value={ fontSize }
            onChange={ ( value ) => setAttributes( { fontSize: value } ) }
            min={ 12 }
            max={ 72 }
          />
        </PanelBody>

        <PanelBody
          title={ __( 'Layout', 'your-text-domain' ) }
          initialOpen={ false }
        >
          <SelectControl
            label={ __( 'Alignment', 'your-text-domain' ) }
            value={ alignment }
            options={ [
              { label: __( 'Left', 'your-text-domain' ), value: 'left' },
              { label: __( 'Center', 'your-text-domain' ), value: 'center' },
              { label: __( 'Right', 'your-text-domain' ), value: 'right' },
            ] }
            onChange={ ( value ) => setAttributes( { alignment: value } ) }
          />
        </PanelBody>
      </InspectorControls>

      <div { ...blockProps } style={ { textAlign: alignment, fontSize: `${ fontSize }px` } }>
        { heading && <h2>{ heading }</h2> }
        { showDate && <p>{ new Date().toLocaleDateString() }</p> }
      </div>
    </>
  );
}
```

### Named sidebar slots with the `group` prop

`InspectorControls` supports a `group` prop to place controls in specific tabs of the sidebar beyond the default Settings tab. This is how core blocks populate the Styles tab panels.

```jsx
import { __ } from '@wordpress/i18n';
import { useBlockProps, InspectorControls } from '@wordpress/block-editor';
import { PanelBody, TextControl } from '@wordpress/components';

export default function Edit( { attributes, setAttributes } ) {
  const blockProps = useBlockProps();
  const { customClass, anchor } = attributes;

  return (
    <>
      { /* Default slot — Settings tab */ }
      <InspectorControls>
        <PanelBody title={ __( 'General', 'your-text-domain' ) }>
          <TextControl
            label={ __( 'Custom CSS Class', 'your-text-domain' ) }
            value={ customClass }
            onChange={ ( value ) => setAttributes( { customClass: value } ) }
          />
        </PanelBody>
      </InspectorControls>

      { /* Advanced tab */ }
      <InspectorControls group="advanced">
        <TextControl
          label={ __( 'HTML Anchor', 'your-text-domain' ) }
          value={ anchor || '' }
          onChange={ ( value ) => setAttributes( { anchor: value } ) }
          help={ __( 'Enter an ID to link directly to this block.', 'your-text-domain' ) }
        />
      </InspectorControls>

      <div { ...blockProps } id={ anchor || undefined } className={ customClass }>
        { __( 'Block content', 'your-text-domain' ) }
      </div>
    </>
  );
}
```

### Color settings panel

`PanelColorSettings` is imported from `@wordpress/block-editor` and renders a styled color section with theme palette support directly inside `InspectorControls`.

```jsx
import { __ } from '@wordpress/i18n';
import {
  useBlockProps,
  InspectorControls,
  PanelColorSettings,
} from '@wordpress/block-editor';

export default function Edit( { attributes, setAttributes } ) {
  const blockProps = useBlockProps();
  const { backgroundColor, textColor } = attributes;

  return (
    <>
      <InspectorControls>
        <PanelColorSettings
          title={ __( 'Color Settings', 'your-text-domain' ) }
          colorSettings={ [
            {
              value: backgroundColor,
              onChange: ( value ) => setAttributes( { backgroundColor: value } ),
              label: __( 'Background Color', 'your-text-domain' ),
            },
            {
              value: textColor,
              onChange: ( value ) => setAttributes( { textColor: value } ),
              label: __( 'Text Color', 'your-text-domain' ),
            },
          ] }
        />
      </InspectorControls>
      <div
        { ...blockProps }
        style={ { backgroundColor, color: textColor, padding: '20px' } }
      >
        { __( 'Styled block content', 'your-text-domain' ) }
      </div>
    </>
  );
}
```

## WordPress Usage (edit.js)

A full realistic `edit.js` combining all three sidebar panels — content settings, typography, and colors — for a feature-rich block.

```jsx
import { __ } from '@wordpress/i18n';
import {
  useBlockProps,
  InspectorControls,
  BlockControls,
  AlignmentControl,
  PanelColorSettings,
  RichText,
} from '@wordpress/block-editor';
import {
  PanelBody,
  ToggleControl,
  RangeControl,
} from '@wordpress/components';

export default function Edit( { attributes, setAttributes } ) {
  const blockProps = useBlockProps();
  const {
    content,
    textAlign,
    showDate,
    fontSize,
    backgroundColor,
    textColor,
  } = attributes;

  return (
    <>
      <BlockControls>
        <AlignmentControl
          value={ textAlign }
          onChange={ ( value ) => setAttributes( { textAlign: value } ) }
        />
      </BlockControls>

      <InspectorControls>
        <PanelBody
          title={ __( 'Content Settings', 'your-text-domain' ) }
          initialOpen={ true }
        >
          <ToggleControl
            label={ __( 'Show Date', 'your-text-domain' ) }
            checked={ showDate }
            onChange={ ( value ) => setAttributes( { showDate: value } ) }
          />
        </PanelBody>

        <PanelBody
          title={ __( 'Typography', 'your-text-domain' ) }
          initialOpen={ false }
        >
          <RangeControl
            label={ __( 'Font Size', 'your-text-domain' ) }
            value={ fontSize }
            onChange={ ( value ) => setAttributes( { fontSize: value } ) }
            min={ 12 }
            max={ 72 }
          />
        </PanelBody>

        <PanelColorSettings
          title={ __( 'Color Settings', 'your-text-domain' ) }
          colorSettings={ [
            {
              value: backgroundColor,
              onChange: ( value ) => setAttributes( { backgroundColor: value } ),
              label: __( 'Background', 'your-text-domain' ),
            },
            {
              value: textColor,
              onChange: ( value ) => setAttributes( { textColor: value } ),
              label: __( 'Text', 'your-text-domain' ),
            },
          ] }
        />
      </InspectorControls>

      <div
        { ...blockProps }
        style={ {
          textAlign,
          fontSize: `${ fontSize }px`,
          backgroundColor,
          color: textColor,
          padding: '1rem',
        } }
      >
        { showDate && <p>{ new Date().toLocaleDateString() }</p> }
        <RichText
          tagName="p"
          value={ content }
          onChange={ ( value ) => setAttributes( { content: value } ) }
          placeholder={ __( 'Write something...', 'your-text-domain' ) }
        />
      </div>
    </>
  );
}
```

## save.js

`InspectorControls` renders only in the editor and produces no front-end output. The `save.js` simply reads the stored attribute values.

```jsx
import { useBlockProps, RichText } from '@wordpress/block-editor';

export default function save( { attributes } ) {
  const blockProps = useBlockProps.save();
  const { content, textAlign, fontSize, backgroundColor, textColor, showDate } = attributes;

  return (
    <div
      { ...blockProps }
      style={ {
        textAlign,
        fontSize: `${ fontSize }px`,
        backgroundColor,
        color: textColor,
        padding: '1rem',
      } }
    >
      { showDate && <p>{ new Date().toLocaleDateString() }</p> }
      <RichText.Content tagName="p" value={ content } />
    </div>
  );
}
```

## Available Groups

| Group | Sidebar location |
|---|---|
| `'default'` | Settings tab — main content area |
| `'advanced'` | Advanced tab |
| `'border'` | Border section in Styles tab |
| `'color'` | Color section in Styles tab |
| `'dimensions'` | Dimensions section in Styles tab |
| `'typography'` | Typography section in Styles tab |

## WordPress Mapping

`InspectorControls` is imported from `@wordpress/block-editor`, not `@wordpress/components`. It is a Slot/Fill component: the editor shell provides the sidebar Slot, and every `InspectorControls` you render in `edit.js` fills into it. You can use multiple `InspectorControls` in the same `edit.js` — one for each `group` value — and their contents will be merged into the appropriate sidebar tab.

Always wrap controls in `PanelBody` for correct visual grouping, expand/collapse behavior, and keyboard accessibility. Use `initialOpen={ false }` on secondary panels to keep the sidebar manageable. For settings that are rarely changed (HTML anchor, custom CSS class), use `group="advanced"` to push them to the Advanced tab instead of cluttering the main Settings panel.

`InspectorControls` must never appear in `save.js`. It has no front-end output; all persisted data travels through block attributes stored in `block.json`.
