# ToggleControl Component

The `ToggleControl` component provides a toggle switch for boolean (true/false) settings.

## Installation

This component is included in the `@wordpress/components` package, which is already available when using `@wordpress/create-block`.

## Required Imports

```javascript
import { ToggleControl } from '@wordpress/components';
```

## block.json Attributes

```json
{
	"attributes": {
		"isEnabled": {
			"type": "boolean",
			"default": false
		}
	}
}
```

## Basic Usage in edit.js

```javascript
import { __ } from '@wordpress/i18n';
import { ToggleControl } from '@wordpress/components';
import { useBlockProps } from '@wordpress/block-editor';

export default function Edit( { attributes, setAttributes } ) {
	const blockProps = useBlockProps();
	const { isEnabled } = attributes;

	return (
		<div { ...blockProps }>
			<ToggleControl
				label={ __( 'Enable Feature', 'your-text-domain' ) }
				checked={ isEnabled }
				onChange={ ( value ) => setAttributes( { isEnabled: value } ) }
			/>
			{ isEnabled && (
				<p>{ __( 'Feature is enabled!', 'your-text-domain' ) }</p>
			) }
		</div>
	);
}
```

import { useBlockProps, InspectorControls } from '@wordpress/block-editor';

	const blockProps = useBlockProps();
	const { isDarkMode, hasAnimation, isShadowEnabled } = attributes;

		<>
			<InspectorControls>
				<PanelBody title={ __( 'Display Settings', 'your-text-domain' ) }>
					<ToggleControl
						label={ __( 'Dark Mode', 'your-text-domain' ) }
						checked={ isDarkMode }
						onChange={ ( value ) => setAttributes( { isDarkMode: value } ) }
					/>
					<ToggleControl
						label={ __( 'Enable Animation', 'your-text-domain' ) }
						checked={ hasAnimation }
						onChange={ ( value ) => setAttributes( { hasAnimation: value } ) }
					/>
					<ToggleControl
						label={ __( 'Enable Shadow', 'your-text-domain' ) }
						checked={ isShadowEnabled }
						onChange={ ( value ) => setAttributes( { isShadowEnabled: value } ) }
					/>
				</PanelBody>
			</InspectorControls>
			<div
				{ ...blockProps }
				className={ `
					${ isDarkMode ? 'dark-mode' : '' }
					${ hasAnimation ? 'animated' : '' }
					${ isShadowEnabled ? 'has-shadow' : '' }
				`.trim() }
			>
				{ __( 'Content with toggleable features', 'your-text-domain' ) }
			</div>
		</>
	);
}
```

import { useBlockProps } from '@wordpress/block-editor';

export default function Edit( { attributes, setAttributes } ) {
	const blockProps = useBlockProps();
	const { isPremiumEnabled } = attributes;

	return (
		<div { ...blockProps }>
			<ToggleControl
				label={ __( 'Premium Feature', 'your-text-domain' ) }
				help={ __( 'This feature requires a premium subscription', 'your-text-domain' ) }
				checked={ isPremiumEnabled }
				onChange={ ( value ) => setAttributes( { isPremiumEnabled: value } ) }
				disabled
			/>
		</div>
	);
}
```

## Props Reference

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | string | - | Label text for the toggle |
| `checked` | boolean | - | Current state of the toggle |
| `onChange` | function | - | Callback when toggle state changes |
| `help` | string | - | Help text displayed below the toggle |
| `disabled` | boolean | `false` | Disable the toggle |
| `className` | string | - | Additional CSS classes |

## Common Use Cases

1. **Visibility Controls**: Show/hide content elements
2. **Feature Flags**: Enable/disable block features
3. **Display Options**: Toggle layout or styling options
4. **Settings**: Boolean configuration options
5. **Conditional Rendering**: Show different content based on state

## Best Practices

- Use clear, descriptive labels
- Provide `help` text for complex options
- Set sensible defaults in `block.json`
- Use conditional rendering based on toggle state
- Group related toggles in PanelBody components
- Consider user expectations (what should default to on/off)

## Accessibility

- Automatically labeled for screen readers
- Keyboard accessible (Space/Enter to toggle)
- Supports focus management
- ARIA attributes handled automatically
- Works with assistive technologies

## WordPress Standards

- Import from `@wordpress/components`
- Use `@wordpress/i18n` for translatable text
- Define boolean attributes in `block.json`
- Use `checked` prop (not `value`)
- Store state in block attributes
