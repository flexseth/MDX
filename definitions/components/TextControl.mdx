# TextControl Component

The `TextControl` component provides a labeled text input field with support for various input types and validation.

## Installation

This component is included in the `@wordpress/components` package, which is already available when using `@wordpress/create-block`.

## Required Imports

```javascript
import { TextControl } from '@wordpress/components';
```

## block.json Attributes

```json
{
	"attributes": {
		"textValue": {
			"type": "string",
			"default": ""
		}
	}
}
```

## Basic Usage in edit.js

```javascript
import { __ } from '@wordpress/i18n';
import { TextControl } from '@wordpress/components';
import { useBlockProps } from '@wordpress/block-editor';

export default function Edit( { attributes, setAttributes } ) {
	const blockProps = useBlockProps();
	const { textValue } = attributes;

	return (
		<div { ...blockProps }>
			<TextControl
				label={ __( 'Enter your text', 'your-text-domain' ) }
				value={ textValue }
				onChange={ ( value ) => setAttributes( { textValue: value } ) }
			/>
		</div>
	);
}
```

## save.js

```javascript
import { useBlockProps } from '@wordpress/block-editor';

export default function save( { attributes } ) {
	const blockProps = useBlockProps.save();
	const { textValue } = attributes;

	return (
		<div { ...blockProps }>
			{ textValue && <p>{ textValue }</p> }
		</div>
	);
}
```

## With InspectorControls (Sidebar)

```javascript
import { __ } from '@wordpress/i18n';
import { TextControl } from '@wordpress/components';
import { useBlockProps, InspectorControls } from '@wordpress/block-editor';
import { PanelBody } from '@wordpress/components';

export default function Edit( { attributes, setAttributes } ) {
	const blockProps = useBlockProps();
	const { heading, subheading } = attributes;

	return (
		<>
			<InspectorControls>
				<PanelBody title={ __( 'Text Settings', 'your-text-domain' ) }>
					<TextControl
						label={ __( 'Heading', 'your-text-domain' ) }
						value={ heading }
						onChange={ ( value ) => setAttributes( { heading: value } ) }
					/>
					<TextControl
						label={ __( 'Subheading', 'your-text-domain' ) }
						value={ subheading }
						onChange={ ( value ) => setAttributes( { subheading: value } ) }
					/>
				</PanelBody>
			</InspectorControls>
			<div { ...blockProps }>
				<h2>{ heading || __( 'Add heading...', 'your-text-domain' ) }</h2>
				<p>{ subheading || __( 'Add subheading...', 'your-text-domain' ) }</p>
			</div>
		</>
	);
}
```

## Props Reference

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | string | - | Label text for the input field |
| `value` | string | - | Current value of the input |
| `onChange` | function | - | Callback when value changes |
| `help` | string | - | Help text displayed below the input |
| `placeholder` | string | - | Placeholder text when input is empty |
| `type` | string | `'text'` | HTML input type: `'text'`, `'email'`, `'tel'`, `'number'`, `'password'`, `'url'` |
| `disabled` | boolean | `false` | Disable the input |
| `className` | string | - | Additional CSS classes |
| `min` | number | - | Minimum value (for type="number") |
| `max` | number | - | Maximum value (for type="number") |
| `step` | number | - | Step value (for type="number") |
| `autoComplete` | string | - | HTML autocomplete attribute |

## Common Use Cases

1. **Simple Text Input**: Collect names, titles, or short text
2. **Email Collection**: Use `type="email"` for email addresses
3. **Phone Numbers**: Use `type="tel"` for phone inputs
4. **Numeric Values**: Use `type="number"` for numbers
5. **Block Settings**: Add to InspectorControls for sidebar settings

## Best Practices

- Always provide a descriptive `label`
- Use `help` text for additional context
- Set appropriate `type` for semantic HTML
- Use `placeholder` for examples, not instructions
- Store values in block attributes with proper defaults

## Accessibility

- Labels are automatically associated with inputs
- Help text is linked via `aria-describedby`
- Supports keyboard navigation
- Works with screen readers

## WordPress Standards

- Import from `@wordpress/components`
- Use `@wordpress/i18n` for translatable text
- Define attributes in `block.json`
- Save attribute values in `save.js`
