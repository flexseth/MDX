# RichText Component

The `RichText` component provides an editable rich text field that supports formatting like bold, italic, links, and more.

## Installation

This component is included in the `@wordpress/block-editor` package, which is already available when using `@wordpress/create-block`.

## Required Imports

```javascript
import { RichText } from '@wordpress/block-editor';
```

## block.json Attributes

```json
{
	"attributes": {
		"content": {
			"type": "string",
			"source": "html",
			"selector": "p"
		}
	}
}
```

## Basic Usage in edit.js

```javascript
import { __ } from '@wordpress/i18n';
import { RichText, useBlockProps } from '@wordpress/block-editor';

export default function Edit( { attributes, setAttributes } ) {
	const blockProps = useBlockProps();
	const { content } = attributes;

	return (
		<RichText
			{ ...blockProps }
			tagName="p"
			value={ content }
			onChange={ ( value ) => setAttributes( { content: value } ) }
			placeholder={ __( 'Enter your text here...', 'your-text-domain' ) }
		/>
	);
}
```

## save.js

```javascript
import { RichText, useBlockProps } from '@wordpress/block-editor';
```

```javascript
}
```

## Props Reference

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `tagName` | string | - | HTML tag to use (p, h1-h6, div, blockquote, etc.) |
| `value` | string | - | Current content value (HTML string) |
| `onChange` | function | - | Callback when content changes |
| `placeholder` | string | - | Placeholder text when empty |
| `allowedFormats` | array | all | Allowed formatting options (see below) |
| `multiline` | string/boolean | `false` | Enable multiline (e.g., 'p', 'li') |
| `className` | string | - | Additional CSS classes |
| `style` | object | - | Inline styles |
| `keepPlaceholderOnFocus` | boolean | `false` | Keep placeholder visible when focused |
| `withoutInteractiveFormatting` | boolean | `false` | Disable link creation |
| `preserveWhiteSpace` | boolean | `false` | Preserve white space and line breaks |

## Common Use Cases

1. **Headings**: Editable heading text (h1-h6)
2. **Paragraphs**: Body content with formatting
3. **Quotes**: Blockquote or pull quotes
4. **Buttons**: Button text (usually no formatting)
5. **Lists**: Ordered or unordered lists
6. **Call-to-Action**: Marketing copy with emphasis

## Best Practices

- Always use appropriate `tagName` for semantic HTML
- Set `allowedFormats={ [] }` for plain text (buttons, labels)
- Use `multiline` for content that needs multiple paragraphs
- Provide descriptive placeholder text
- Match edit.js and save.js tag names exactly
- Use `source: "html"` in block.json attributes
- Include `selector` in block.json to match tagName

## Accessibility

- Use semantic HTML tags (h1-h6, p, blockquote)
- Don't skip heading levels
- Ensure sufficient color contrast
- Test with screen readers
- Support keyboard navigation (built-in)

## WordPress Standards

- Import from `@wordpress/block-editor`
- Use `RichText.Content` in save.js (not RichText)
- Use `@wordpress/i18n` for translatable placeholders
- Define HTML source attributes in `block.json`
- Always specify `selector` for proper content extraction
