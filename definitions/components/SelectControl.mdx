# SelectControl Component

The `SelectControl` component provides a dropdown select menu for choosing from a list of predefined options.

## Installation

This component is included in the `@wordpress/components` package, which is already available when using `@wordpress/create-block`.

## Required Imports

```javascript
import { SelectControl } from '@wordpress/components';
```

## block.json Attributes

```json
{
	"attributes": {
		"alignment": {
			"type": "string",
			"default": "left"
		}
	}
}
```

## Basic Usage in edit.js

```javascript
import { __ } from '@wordpress/i18n';
import { SelectControl } from '@wordpress/components';
import { useBlockProps } from '@wordpress/block-editor';

export default function Edit( { attributes, setAttributes } ) {
	const blockProps = useBlockProps();
	const { alignment } = attributes;

	return (
		<div { ...blockProps }>
			<SelectControl
				label={ __( 'Alignment', 'your-text-domain' ) }
				value={ alignment }
				options={ [
					{ label: __( 'Left', 'your-text-domain' ), value: 'left' },
					{ label: __( 'Center', 'your-text-domain' ), value: 'center' },
					{ label: __( 'Right', 'your-text-domain' ), value: 'right' },
				] }
				onChange={ ( value ) => setAttributes( { alignment: value } ) }
			/>
			<p style={ { textAlign: alignment } }>
				{ __( 'Aligned text', 'your-text-domain' ) }
			</p>
		</div>
	);
}
```

## Post Type Selector

```javascript
import { __ } from '@wordpress/i18n';
import { SelectControl } from '@wordpress/components';
import { useBlockProps } from '@wordpress/block-editor';
import { useSelect } from '@wordpress/data';

export default function Edit( { attributes, setAttributes } ) {
	const blockProps = useBlockProps();
	const { postType } = attributes;

	const postTypes = useSelect( ( select ) => {
		const types = select( 'core' ).getPostTypes( { per_page: -1 } );
		return types
			? types
				.filter( ( type ) => type.viewable )
				.map( ( type ) => ( {
					label: type.name,
					value: type.slug,
				} ) )
			: [];
	}, [] );

	return (
		<div { ...blockProps }>
			<SelectControl
				label={ __( 'Post Type', 'your-text-domain' ) }
				value={ postType }
				options={ postTypes }
				onChange={ ( value ) => setAttributes( { postType: value } ) }
			/>
		</div>
	);
}
```

## Props Reference

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | string | - | Label text for the select control |
| `value` | string | - | Currently selected value |
| `onChange` | function | - | Callback when selection changes |
| `options` | array | - | Array of option objects with `label` and `value` |
| `help` | string | - | Help text displayed below the control |
| `disabled` | boolean | `false` | Disable the select control |
| `className` | string | - | Additional CSS classes |
| `children` | node | - | For advanced usage with optgroup |

## Option Object Format

```javascript
{
	label: 'Display Text',  // Text shown to user
	value: 'value'          // Value stored in attribute
}
```

## Common Use Cases

1. **Alignment**: Text/content alignment options
2. **Layout**: Choose between layout types
3. **Styling**: Theme, color scheme, or style variants
4. **Post Type**: Select WordPress post types
5. **Taxonomy**: Choose categories or tags
6. **Size/Scale**: Predefined size options

## Best Practices

- Provide clear, descriptive labels
- Use consistent option naming
- Set sensible defaults in `block.json`
- Consider using help text for complex options
- Order options logically (alphabetical, by frequency, etc.)
- For many options (>10), consider using a different control
- Use string values in attributes, convert to numbers if needed

## Accessibility

- Labels automatically associated with selects
- Keyboard navigable (arrow keys, Enter, Escape)
- Screen reader friendly
- ARIA attributes handled automatically
- Focus management built-in

## WordPress Standards

- Import from `@wordpress/components`
- Use `@wordpress/i18n` for translatable text
- Define string attributes in `block.json`
- Use `useSelect` from `@wordpress/data` for dynamic options
- Follow block editor design patterns
