# Guide Component

The `Guide` component from `@wordpress/components` renders a multi-page tutorial inside a modal. Users can navigate forward and backward through pages, making it ideal for onboarding flows, feature introductions, and step-by-step instructions.

## Installation

This component is included in the `@wordpress/components` package, which is already available when using `@wordpress/create-block`.

## Required Imports

```javascript
import { Guide } from '@wordpress/components';
```

## Basic Usage in edit.js

```javascript
import { __ } from '@wordpress/i18n';
import { Guide, Button } from '@wordpress/components';
import { useBlockProps } from '@wordpress/block-editor';
import { useState } from '@wordpress/element';

export default function Edit() {
	const blockProps = useBlockProps();
	const [ isOpen, setIsOpen ] = useState( false );

	return (
		<div { ...blockProps }>
			<Button
				variant="primary"
				onClick={ () => setIsOpen( true ) }
			>
				{ __( 'Open Guide', 'your-text-domain' ) }
			</Button>

			{ isOpen && (
				<Guide
					contentLabel={ __( 'Welcome Guide', 'your-text-domain' ) }
					finishButtonText={ __( 'Get Started', 'your-text-domain' ) }
					onFinish={ () => setIsOpen( false ) }
					pages={ [
						{
							content: (
								<p>
									{ __( 'Welcome! This guide will walk you through the basics.', 'your-text-domain' ) }
								</p>
							),
						},
						{
							content: (
								<p>
									{ __( 'You have completed the guide. Click "Get Started" to begin.', 'your-text-domain' ) }
								</p>
							),
						},
					] }
				/>
			) }
		</div>
	);
}
```

## Props Reference

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | string | - | Custom CSS class added to the Guide modal |
| `contentLabel` | string | **Required** | Accessible label for the modal (used by screen readers) |
| `finishButtonText` | string | `'Finish'` | Text for the button on the last page |
| `nextButtonText` | string | `'Next'` | Text for the forward navigation button |
| `previousButtonText` | string | `'Previous'` | Text for the backward navigation button |
| `onFinish` | function | **Required** | Callback when the guide is closed or the finish button is clicked |
| `pages` | array | `[]` | Array of page objects defining each step |

## Page Object Properties

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `content` | ReactNode | Yes | The main content for the page |
| `image` | ReactNode | No | An optional image displayed above the content |

## Common Use Cases

1. **Plugin Onboarding**: Welcome new users with a guided tour after activation
2. **Feature Introductions**: Highlight new features after a plugin update
3. **Setup Wizards**: Walk users through initial configuration steps
4. **Block Tutorials**: Explain how to use a custom block within the editor

## Accessibility

- Always provide a descriptive `contentLabel` for screen readers
- The modal traps focus while open, supporting keyboard navigation
- Page indicators (dots) are rendered for visual progress tracking
- Previous/Next buttons are keyboard accessible by default

## WordPress Standards

- Import from `@wordpress/components`
- Use `@wordpress/i18n` for all translatable strings
- Use `@wordpress/element` for `useState` and other React hooks
- Use icons from `@wordpress/icons` for visual elements within pages
- Follow block editor design patterns for consistent UI
